# LVGL Pro Editor — Workflow Guide
## Cerberus Dashboard (480×320)

Panduan ini menjelaskan cara mendesain UI dengan **LVGL Pro Editor** dan
mengintegrasikannya ke desktop simulator maupun firmware ESP32.

---

## Arsitektur

```
project/
├── desktop_sim/          ← Simulator (LVGL + SDL2, cmake)
│   ├── main.cpp          ← Event loop SDL2 + LVGL
│   ├── mock_data.cpp/h   ← Data palsu untuk testing
│   ├── lv_conf.h         ← Konfigurasi LVGL
│   └── build.sh          ← Script build & run
│
├── ui/                   ← UI layer (bisa di-generate Pro Editor)
│   ├── dashboard_ui.h    ← Interface: init() + update()
│   └── dashboard_ui.c    ← Implementasi widget LVGL
│
└── src/                  ← Firmware ESP32 (PlatformIO)
    └── ...
```

---

## A. Menjalankan Simulator (tanpa Pro Editor)

```bash
cd desktop_sim
./build.sh          # build + jalankan (download LVGL otomatis ~40MB)
./build.sh build    # hanya build
./build.sh run      # hanya jalankan
./build.sh clean    # bersihkan build folder
```

**Keys di window simulator:**

| Key | Mode |
|-----|------|
| `1` | RPM sweep (0–6000 RPM sinusoidal) |
| `2` | Idle (850 RPM) |
| `3` | Driving (1500–4000 RPM) |
| `4` | Redline (5700+ RPM) |
| `0` | Data off |
| `Q` / `ESC` | Quit |

## B. Workflow LVGL Pro Editor

### 1. Setup Proyek di Editor

1. Download **LVGL Pro Editor** dari [lvgl.io/pro-editor](https://lvgl.io/pro-editor)
2. Buat proyek baru:
   - **Resolution:** `480 × 320`
   - **Color depth:** `32 bit`
   - **Theme:** Dark
3. Set **Export path** ke folder `ui/` di project ini

### 2. Desain Dashboard

Buat widget dengan nama yang sudah ditentukan agar `dashboard_ui_update()` bisa
map data dengan benar:

| Widget | Name di Editor | Type |
|--------|---------------|------|
| RPM gauge | `ui_RPMArc` | Arc |
| RPM angka | `ui_RPMLabel` | Label |
| AFR value | `ui_AFRLabel` | Label |
| TPS value | `ui_TPSLabel` | Label |
| MAP value | `ui_MAPLabel` | Label |
| CLT value | `ui_CLTLabel` | Label |
| BAT value | `ui_BATLabel` | Label |
| SIM badge | `ui_SimBadge` | Label |

### 3. Export dari Editor

- **File → Export → C files**
- Pilih folder: `ui/`
- Editor akan generate:
  ```
  ui/
  ├── ui.h               ← include utama
  ├── ui.c               ← ui_init() + screen list
  ├── ui_Screen1.c       ← widget definitions
  ├── ui_helpers.h       ← macro helpers
  └── fonts/             ← font files jika ada custom font
  ```

### 4. Update `dashboard_ui.c` untuk Export

Setelah export, update `ui/dashboard_ui.c` agar menggunakan variabel dari editor:

```c
#include "ui.h"   // generated by LVGL Pro Editor

// Variabel widget di-export oleh editor, misal:
// extern lv_obj_t *ui_RPMArc;
// extern lv_obj_t *ui_RPMLabel;
// dll.

void dashboard_ui_init(void)
{
    ui_init();   // panggil fungsi init yang di-generate editor
}

void dashboard_ui_update(const DashboardData *d)
{
    // Map data ke widget yang di-export editor
    lv_arc_set_value(ui_RPMArc, d->rpm);
    lv_label_set_text_fmt(ui_RPMLabel, "%d", d->rpm);
    lv_label_set_text_fmt(ui_AFRLabel, "%.1f", d->afr);
    lv_label_set_text_fmt(ui_TPSLabel, "%.1f", d->tps);
    lv_label_set_text_fmt(ui_MAPLabel, "%.0f", d->map_kpa);
    lv_label_set_text_fmt(ui_CLTLabel, "%.0f", d->coolant_temp);
    lv_label_set_text_fmt(ui_BATLabel, "%.1f", d->battery_voltage);

    if (d->simulator_active)
        lv_obj_clear_flag(ui_SimBadge, LV_OBJ_FLAG_HIDDEN);
    else
        lv_obj_add_flag(ui_SimBadge, LV_OBJ_FLAG_HIDDEN);
}
```

### 5. Rebuild Simulator

```bash
cd desktop_sim
./build.sh
```

## C. Transfer UI ke ESP32 (Firmware)

LVGL Pro Editor bisa juga generate kode untuk ESP32, **tapi** firmware ini
menggunakan **TFT_eSPI** (bukan LVGL). Ada dua jalur:

### Jalur A: Tetap pakai TFT_eSPI (saat ini)
- Desktop sim menggunakan LVGL hanya untuk desain/preview
- UI production tetap ditulis manual dengan TFT_eSPI draw calls
- Cocok untuk firmware yang sudah ada

### Jalur B: Migrasi firmware ke LVGL (rekomendasi untuk masa depan)
1. Tambahkan library LVGL ke `platformio.ini`:
   ```ini
   lib_deps =
       lvgl/lvgl @ ^9.2.0
   ```
2. Buat `src/lv_port_disp.cpp` — wrapper TFT_eSPI → LVGL flush_cb
3. Copy folder `ui/` ke project (shared antara sim dan firmware)
4. Ganti `drawData()` dengan `lv_timer_handler()` di loop

## D. Tips

- **Hot-reload desain:** Edit `dashboard_ui.c` → `./build.sh` (LVGL tidak perlu
  di-download ulang, hanya recompile file UI)
- **Test data ekstrem:** Tekan `4` (redline) untuk cek warning color RPM merah
- **Custom font:** Tambahkan ke `lv_conf.h` bagian `LV_FONT_MONTSERRAT_*`
- **Ganti resolusi:** Ubah `DISPLAY_WIDTH/HEIGHT` di `main.cpp` dan di editor
